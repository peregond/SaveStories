## SaveStories 0.4.30

### Изменения с 0.4.20 по 0.4.30

#### 0.4.20

- Добавлен выбор, что сохранять: только видео или фото и видео.
- Добавлено отображение прогресса в реальном времени по файлам и папкам.
- Улучшена пакетная выгрузка и общая стабильность worker runtime.

#### 0.4.21

- Новый экран `Главная` стал основным сценарием для пакетной выгрузки.
- Добавлены недавние наборы профилей для повторного запуска.
- Подготовлен отдельный раздел `Reels` как следующий этап развития интерфейса.

#### 0.4.22

- Улучшены release-сборки macOS: исправлена упаковка `Sparkle.framework` и fallback на встроенный Python runtime.
- Исправлено сохранение пользовательской папки выгрузки между перезапусками.
- Исправлены статусы пакетной очереди и корректный подсчёт обработанных профилей.
- Для Windows сохранена совместимость настроек после переименования приложения.

#### 0.4.25

- Полностью переработан интерфейс `Главной` на macOS: блоки `Status`, `Result`, `Fast Start`, `Недавнее`, `Очередь` и `Логи` собраны в новый рабочий layout.
- Улучшен узкий layout окна: экран дольше остаётся двухколоночным и аккуратнее перестраивается на маленькой ширине.
- Блок `Недавнее` показывает последний набор и умеет раскрывать остальные через кнопку `ещё`.
- В `Status` добавлены более живые индикаторы: светящаяся зелёная точка, анимация текущего шага и live-обновление шага по ходу пакетной выгрузки.
- Обновлён звук запуска кнопки `Скачать`, а подписи и порядок действий в `Fast Start` стали компактнее и понятнее.
- Версия синхронизирована для macOS и Windows, чтобы обе платформы можно было выпустить как единый релиз `0.4.25`.

#### 0.4.26

- Подтянут UI-паритет `Главной` для Windows: в `Быстрый старт` теперь компактные действия `Добавить / Запомнить / Очистить` и основной запуск `Скачать`.
- В правой колонке Windows добавлен блок `Недавнее` с режимом просмотра только последнего набора и кнопкой `Ещё` для раскрытия остальных.
- В `Главной` Windows добавлен встроенный блок логов и синхронизация live-лога прямо на экране запуска.
- Для Windows-апдейтера улучшен выбор release-установщика: поддерживаются варианты имён с/без `v`, добавлен fallback на `SaveStories-Windows-Setup-*.exe` и более подробные диагностические сообщения.

#### 0.4.27

- Вынесены отдельные слои для `batch`, `update` и `runtime/session` логики на macOS и Windows, чтобы дальнейшие изменения было проще сопровождать и проверять.
- Добавлен smoke CI для `swift build`, `swift test`, Node syntax-check и компиляции Windows Python-кода.
- Добавлены первые Swift-тесты для `batch`, `runtime` и `AppPaths`, чтобы закрепить критичную логику перед следующими релизами.

#### 0.4.28

- В Windows `Главная` снова собрана в две колонки, без отдельной третьей панели `Активность`, чтобы совпадать с macOS.
- Для Windows и Node добавлены первые unit-тесты, а `main.py` дополнительно разрезан по runtime/request flow.
- В smoke CI добавлены Node tests, Windows Python tests и проверка согласованности версии между релизными файлами.

#### 0.4.29

- Исправлен hotfix для Node worker: выгрузка stories больше не падает с `ReferenceError: mediaVariantTag is not defined` на профилях с видео-story.

#### 0.4.30

- Исправлен ещё один hotfix в Node worker: после выноса media-утилит восстановлен импорт `isAudioOnlyVariant`, чтобы избежать второго скрытого runtime-падения на видео-variants.
