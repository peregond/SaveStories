# SaveMe WinUI — Gap до финальной стабильной Windows-версии

## Что уже закрыто

- Stories / Queue / Reels подключены к `node_worker` и работают через общий bridge.
- Есть проверка сессии, запуск/остановка задач, ретрай неудачных элементов.
- Есть встроенный апдейтер с установкой `exe` поверх текущей версии.
- Есть установка runtime-модулей (post-install) и базовая диагностика.
- Есть выбор темы (`Системная / Светлая / Тёмная`) и сохранение настройки.

## Что ещё не хватает до stable parity с legacy Windows-клиентом

### Must-have (рекомендовано закрыть до снятия `.beta`)

1. Недавние наборы профилей для Stories:
- `Запомнить` текущий набор.
- Список недавних наборов справа.
- Действия `Добавить в очередь` / `Заменить очередь` / удаление набора.
- Персистентность в настройках.

2. Управление сном во время выгрузки:
- Тумблер в настройках `Не давать компьютеру засыпать`.
- Реальная блокировка сна ОС только на время активной выгрузки.

3. Пост-обработка пустых папок (Stories):
- После завершения batch предлагать удаление пустых папок.
- Показ кандидатов на удаление (раскрываемый список).
- Отчёт по факту удаления.

### Should-have (желательно для полного UX-паритета)

1. Выбор папки через нативный folder picker (не только ручной ввод пути).
2. Цветная классификация логов + кнопка копирования с подтверждением.
3. Более детальные статус/результат карточки (как в macOS/legacy версии).
4. Улучшенный экран Settings: состояния служб, более полный flow подключения к Instagram.

### Nice-to-have

1. Расширенные UI-анимации статуса/очереди.
2. Тонкая полировка адаптивности для маленькой высоты окна.

## План релизной стабилизации

1. Закрыть все `Must-have` пункты.
2. Прогнать smoke-тесты Stories/Queue/Reels/Updater на чистой Windows-машине.
3. Выпустить как стабильный канал с единым инсталлятором `SaveMe-Windows-Setup-vX.Y.Z.exe`.
4. Сохранить обратную совместимость апдейтера с предыдущими beta-asset именами.
